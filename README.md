# system-generator-for-DSP-flow


## Responsabilités liées à la publication du code source

Ce projet est hébergé sur GitHub afin d’assurer la traçabilité, le contrôle de versions et la transparence des différentes itérations de conception liées à la modélisation de systèmes DSP à l’aide de System Generator.

En tant qu’auteur du dépôt, mes responsabilités incluent :

Le maintien d’une structure claire et cohérente du dépôt

La publication de fichiers correspondant uniquement à des travaux de laboratoire validés

L’utilisation de messages de commit explicites pour documenter l’évolution du projet

La prévention de modifications non autorisées des fichiers de référence académiques

Différentes stratégies de diffusion du code peuvent être envisagées avec GitHub :

Dépôt privé pour le développement initial ou les contenus sensibles

Dépôt public pour le partage académique, la reproductibilité et la validation par les pairs

Versions figées (tags / releases) pour identifier des états stables et validés du projet

Dans le cadre de ce projet, un dépôt public sans releases formelles a été choisi, l’objectif principal étant l’évaluation académique et non le déploiement logiciel.

## Gestion des dysfonctionnements et corrections (bug fixes)

En cas de dysfonctionnement du code ou d’erreurs de conception :

Les problèmes sont identifiés à partir des résultats de simulation ou de comportements inattendus dans System Generator

Les corrections sont apportées directement dans les modèles .slx

Chaque correction fait l’objet d’un nouveau commit Git afin d’assurer la traçabilité

Les versions précédentes restent accessibles, permettant un retour arrière si nécessaire

Cette approche reflète les bonnes pratiques d’ingénierie, où le contrôle de versions est essentiel pour le débogage et la validation des systèmes.

## Choix de la licence et justification

Ce projet est distribué sous la licence MIT.

La licence MIT a été choisie car :

Elle autorise librement l’utilisation, la modification et la redistribution du code

Elle est simple, claire et largement utilisée dans les milieux académiques et industriels

Elle est adaptée aux projets pédagogiques ne visant pas une exploitation commerciale

Elle limite la responsabilité de l’auteur en cas de mauvaise utilisation ou de dysfonctionnement du code

Ce choix de licence est cohérent avec l’objectif académique du projet : partager les connaissances tout en définissant clairement les responsabilités.
